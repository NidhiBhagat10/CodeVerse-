HTML(Hypertext Markup Language)

  • Structure of Web Content: HTML is the standard markup language used to create the structure of web content. It consists of a set of elements and tags that define the different parts of a web page, such as headings, paragraphs, images, links, forms, and more.

  • Hierarchical Structure: HTML documents are organized hierarchically using nested elements. Elements are often structured as opening tags, content, and closing tags (e.g., <tag>content<⁄tag>). The hierarchy defines the relationships between different parts of the content.

  • Semantic Markup: HTML supports semantic markup, which means that elements are chosen based on their meaning and purpose. This helps search engines, accessibility tools, and other software understand the content better, enhancing user experience and SEO.

HTML Structure & Hierarchy

HTML is usually structured as a set of family tree relationships.

<body>
<header>
<h1>My recent trip to Greece<⁄h1>
<⁄header>

<main>
<h2>Overview<⁄h2>
<h3>
<ol>
<li>Greece<⁄li>
<li>Trip Itinerary<⁄li>
<⁄ol>
<⁄h3>

In the sample above,
  • <h2>Overview<⁄h2> is the child of<main>
  • <li>Greece<⁄li> and <li> Trip Itinerary<⁄li> are at the same level - they are siblings
  • Both <header> and <main> are the children of <body>
HTML hierarchy and its importance will be relevant when we learn CSS.

Child elements inherit behaviour and styling from their parent element.

<div> is a powerful element in HTML

It is the 'The one ring to rule them all'.

  • <div>is a container that divides the page into sections
  • <div>helps to apply CSS styles to a block of content.
  • <div>element can hold various types of content,including text, images, videos, tables.
<div>elements were primary used to create groups or sections which have a common logical and visual behaviour on our webpage.

In Semantic HTML - instead of using<div> - we use tags such as <header>, <nav>, <section> ,<article>, <aside>.
  • Once written in semantic html, the code is easier to read for other developers
  • The code is also easier to read for screen readers and for SEO since they are able to under stand what each section of the code is supposed to represent
  • But what exactly is the difference you ask?Visually - this is how different blocks of code would get called out using semantic html
Whenever we use HTML to set up our webpage, we will use both non-semantic HTML (div) and Semantic HTML.

Semantic HTML

Semantic HTML - Header and footer

The <header>semantic element is used for the main title or introductory element - usually the <h1> or <h2> tag at the top of the page.

The <footer> element is used for the bottom of the webpage usually containing information such as 'Contact us', Terms and conditions', 'About us', etc.

Sample syntax:

<header>
<h1>
Everything you need to know about pizza!
<⁄h1>
<⁄header>
<footer>
<p> All rights reserved.Copyright@hrishik85 <⁄p>
<⁄footer>

Semantic HTML - Main, Section & Article

Usually- the <header> and <footer> elements fall outside the <main> element.

<main> will contain the primary content of the webpage.

  • <main> will contain semantic elements such as <section> ,<article> and non-semantic elements such as<div>
  • <section> is usually used for a heading and paragraphs associated with the heading or a chapter
  • <article> usually used for a self-contained composition such as comments or excerpts
The syntax format is as follows
