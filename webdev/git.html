Git and GitHub

Git is a very popular version control system , it is a tool that helps to track changes in code or history of our project.

Used to -

  • track the history
  • To colaborate.

GitHub is a Website that allows developer to sgtore and manage their code using git. Project folders are called Repository.

Git

git --version - Shows git downloged version if installed properly. From here we can confirm that if git is installed in our system or not.
ls - list all the files and directoried.
ls -a - shows all files with hidden files.

If it shows .git it means this file is tracked by git.
clear - clear command window.
pwd - find our current working directory.

Configuring Git

Telling your git in which account you are going to change (change from git to gitHub).

git config --global user.name "Your Name"
global level - pure system pe change(by one account) , Local level - in specific project or repo with different account.
git config --global user.email "someone@email.com"
git config --list
~ - shows we are in systems root directory/main directory

Clone & Status

Clone - cloning a repository on our local machine

git clone <- some link ->
remote - github, local - laptop/PC
Status - Display the status of the code
git status
status type -
Untracked - new files that git dosen't yet track
modified - changed
staged - file is ready to be committed
unmodified - unchanged

cd <-foldername-> - change directory, to go from outer folder to inner folder.
cd .. - back to outer folder or directory.
mkdir - make directory, used to make new directory.

Add & Commit

add - adds new or changed files in your working directory to the Git staging area.

git add <-file name->
Commit - it is the record of change
git commit -m"some message"

Push Command

push - upload local repo content to remote repo

git push origin main

init Command

init - used to creat a new git repo. to initialize git in an Existing project in local machine.

git init
git remote add origin <-link->
git remote -v (to verify remote)
git branch (to check branch)
git branch -M main (to rename branch)
git push origin main
